# Internationalization (i18n)

> Cómo funciona la internacionalización en Storico.
> Última actualización: 2026-07-15

## Stack

- **Framework**: Astro i18n nativo
- **Locales**: `en` (inglés, default), `es` (español)
- **Estrategia de routing**: `pathname` con prefijo en todos los locales

## Configuración

En `astro.config.mjs`:

```javascript
i18n: {
  defaultLocale: 'en',
  locales: ['en', 'es'],
  routing: {
    prefixDefaultLocale: true,      // /en/dashboard, /es/dashboard
    strategy: 'pathname',           // locale en el path
  },
}
```

## Cómo funciona

### Detección de idioma

```typescript
// Detecta de header Accept-Language
detectLocale(acceptLanguage: string): 'en' | 'es'
```

### Uso en componentes React

Las islas React reciben el locale como prop desde la página Astro:

```tsx
// Página .astro
<StoryList client:load locale={Astro.currentLocale} />

// Componente React
function StoryList({ locale }: { locale: string }) {
  const t = useTranslations(locale);
  return <h1>{t('stories.title')}</h1>;
}
```

### Uso en páginas Astro

```astro
---
const t = useTranslations(Astro.currentLocale);
---
<h1>{t('landing.hero.title')}</h1>
```

### Rutas

```typescript
// Generar path con locale
localizedPath(path: string, locale: string): string
// /en/dashboard, /es/dashboard

// Extraer locale de la URL
getLocaleFromPath(pathname: string): 'en' | 'es'
```

### Páginas públicas (sin auth)

```typescript
isPublicPagePath(pathname: string): boolean
// /, /login, /about, /privacy, /terms, /docs, /api, /status
```

### Páginas protegidas (requieren auth)

```typescript
isProtectedPagePath(pathname: string): boolean
// /dashboard, /stories, /kanban, /export, /account
```

## Reglas

1. **User stories siempre en inglés** — El input del usuario debe seguir formato INVEST en inglés porque los LLMs responden mejor.
2. **Tareas generadas en inglés** — Output del LLM.
3. **Prompts del sistema en español** — Como se definió en la tesis.
4. **UI traducible** — Labels, botones, mensajes de error, tooltips.
5. **Sin texto hardcodeado** — Todo texto visible pasa por `useTranslations()`.

## Agregar un nuevo idioma

1. Agregar el locale a `astro.config.mjs` → `i18n.locales`
2. Crear `frontend/src/i18n/{locale}.json` con todas las claves traducidas
3. Agregar el locale a `detectLocale()` en `src/i18n/utils.ts`
4. Agregar a `Locale` type en `src/i18n/utils.ts`
